# 校园二手交易平台数据库系统主线实施计划

> 面向执行者：本文件是当前项目的主线执行计划。必须按任务顺序推进；每完成一个任务，都要先更新 `mainline_status.md`，再进入下一个任务。

**目标：** 交付一个可在线访问的校园二手交易平台数据库系统，完整覆盖 `request.md` 中的建库、数据操作、查询、视图、购买逻辑、说明文档与演示要求。

**架构：** 采用 Flask + SQLite + Jinja2 构建单体 Web 应用。数据库结构与初始化数据通过 SQL 脚本维护，页面直接展示数据库结果；购买商品使用事务保证订单写入与商品状态更新的一致性。部署阶段优先选择能够快速提供公网访问地址的平台。

**技术栈：** Python 3.x、Flask、SQLite、Jinja2、HTML/CSS、SQL、可选 `pytest`

---

## 文件规划

- `request.md`：原始作业需求，执行过程中不可偏离。
- `mainline_plan.md`：主线计划，维护任务拆分、执行顺序与验收标准。
- `mainline_status.md`：主线状态，记录每个任务的当前状态、阻塞与日志。
- `app.py`：Flask 应用入口，负责路由与页面渲染。
- `db/schema.sql`：数据库建表语句、约束与视图定义。
- `db/seed.sql`：题目初始数据与演示用补充数据。
- `db/queries.sql` 或 `services/queries.py`：题目要求的查询与聚合逻辑。
- `templates/index.html`：首页与导航。
- `templates/items.html`：商品列表页，以及查询、增删改、购买入口。
- `templates/users.html`：用户列表页与用户发布统计展示。
- `templates/orders.html`：订单列表页与连接查询结果展示。
- `static/style.css`：页面样式。
- `README.md` 或 `docs/report.md`：提交用说明文档，包含部署、截图、简答与提交说明。

## 执行约定

- 每次只能有 1 个任务处于「进行中」状态。
- 开始任务前，先将 `mainline_status.md` 中对应任务标记为「进行中」。
- 完成任务后，必须先做该任务对应验证，再将状态改为「已完成」。
- 若任务范围、文件路径或技术方案发生变化，必须同步更新 `mainline_plan.md` 和 `mainline_status.md`。
- 若出现阻塞，必须在 `mainline_status.md` 的「当前阻塞」和「更新日志」中记录原因与下一步。
